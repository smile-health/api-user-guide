## About

This document aims to provide a detailed explanation of the access rights assigned to different roles 
within the Transaction Service in SMILE. In this service, there are four roles with varying levels of 
access: Super Admin, Admin, Manager, and Operator.

Similar to other services, access to the system’s platform is role-dependent. The Super Admin and 
Admin roles can only access the system through the Web App, whereas the Manager role has access 
to both the Web and Mobile platforms. Meanwhile, the Operator role is restricted to accessing the 
system via the Mobile platform only because this role’s focus is on-field operational process.

Additionally, each role has specific permissions that define whether they have full control over certain 
features or are limited to view-only access. To ensure proper authorization and security, the Role 
Matrix documentation serves as a structured guideline for managing user permissions effectively. 
It helps maintain system integrity, prevent unauthorized actions, and ensure that each role operates 
within its designated scope. By clearly defining access levels, this document also supports compliance, 
operational efficiency, and overall system governance. This structured approach ensures clarity in 
access levels and enhances the management across the system.
